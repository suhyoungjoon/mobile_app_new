# 📄 보고서 파일 예시 가이드

## 📋 개요

이 문서는 InsightI에서 생성되는 보고서 파일의 예시를 보여줍니다.

---

## 📁 보고서 파일 형식

InsightI는 다음과 같은 형식의 보고서를 생성합니다:

1. **HTML 보고서**: 웹 브라우저에서 확인 가능한 HTML 형식
2. **PDF 보고서**: 인쇄 및 공유를 위한 PDF 형식 (향후 구현 예정)

---

## 🎨 보고서 구조

### 1. 헤더
- 보고서 제목: "세대 점검 종합보고서"
- 부제목: "InsightI Pre/Post Inspection Comprehensive Report"

### 2. 점검 정보
- 단지명
- 동-호
- 세대주
- 점검일
- 점검 유형
- 보고서 생성일

### 3. 점검 요약
- 총 하자 건수
- 열화상 점검 건수
- 공기질 측정 건수
- 라돈 측정 건수
- 레벨기 측정 건수
- 총 장비점검 건수

### 4. 하자 상세 목록
각 하자는 카드 형식으로 표시되며 다음 정보를 포함합니다:
- 하자 번호
- 위치 / 세부공정
- 하자 내용
- 메모 (있는 경우)
- 사진 (근거리/원거리)

### 5. 장비점검 결과 (점검원 권한 사용자)
- 공기질 측정 결과 (테이블)
- 라돈 측정 결과 (테이블)
- 레벨기 측정 결과 (테이블)
- 열화상 점검 결과 (카드 형식)

### 6. 푸터
- 보고서 생성일
- 비고 사항

---

## 📄 보고서 예시 파일

### HTML 예시
**파일**: `REPORT_EXAMPLE.html`

이 파일은 실제 보고서의 HTML 형식을 보여주는 예시입니다.

**포함된 내용**:
- 점검 정보 (단지명, 동-호, 세대주 등)
- 점검 요약 (하자 3건, 장비점검 4건)
- 하자 상세 목록 (3개 하자 카드)
  - 하자 #1: 거실 / 바닥재 - 마루판 들뜸
  - 하자 #2: 주방 / 타일 - 타일 균열
  - 하자 #3: 욕실 / 타일 - 욕실 곰팡이

**확인 방법**:
1. 브라우저에서 `REPORT_EXAMPLE.html` 파일을 엽니다
2. 또는 다음 명령어로 열 수 있습니다:
   ```bash
   open REPORT_EXAMPLE.html  # macOS
   # 또는
   start REPORT_EXAMPLE.html  # Windows
   # 또는
   xdg-open REPORT_EXAMPLE.html  # Linux
   ```

## 🎨 보고서 디자인 특징

### 색상 스키마
- **주요 색상**: #1a73e8 (파란색)
- **요약 배경**: #e8f5e8 (연한 녹색)
- **메모 배경**: #f0f8ff (연한 파란색)

### 레이아웃
- **최대 너비**: 1000px
- **여백**: 40px (모바일에서는 자동 조정)
- **카드 스타일**: 그림자 효과, 둥근 모서리

### 반응형 디자인
- 모바일, 태블릿, 데스크톱에서 모두 최적화
- 인쇄 시 레이아웃 자동 조정

## 📥 보고서 생성 방법

### 1. 보고서 미리보기
- 앱에서 "보고서" 탭 클릭
- HTML 형식으로 화면에 표시

### 2. PDF 생성 (향후 구현 예정)
- 보고서 미리보기 화면에서 "PDF 다운로드" 버튼 클릭
- PDF 파일이 다운로드됨

### 3. 보고서 발송
- "문자 발송(모의)" 버튼 클릭
- 전화번호 입력
- 보고서 링크가 SMS로 발송됨

## 💡 참고사항

### 보고서에 포함되지 않는 내용
- 실제 사진 파일 (보고서에는 사진 플레이스홀더만 표시)
- 원본 측정 데이터 파일
- 추가 첨부 파일

### 보고서 생성 시 주의사항
- 보고서는 최신 케이스 데이터를 기반으로 생성됩니다
- 하자가 없는 경우 "등록된 하자가 없습니다" 메시지가 표시됩니다
- 장비점검 데이터는 점검원 권한 사용자만 확인할 수 있습니다
