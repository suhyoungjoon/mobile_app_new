## 🚀 시작하기

### 접속 방법
- **웹 주소**: https://insighti.vercel.app
- **모바일 최적화**: 스마트폰 브라우저에서 접속 시 자동으로 모바일 화면으로 전환됩니다.
- **PWA 설치**: 홈 화면에 추가하여 앱처럼 사용할 수 있습니다.

### 로그인
1. 화면에서 **단지명**, **동**, **호**, **성명**, **전화번호**를 입력합니다.
2. **"로그인(3일 사용권)"** 버튼을 클릭합니다.
3. 로그인 성공 시 메인 화면으로 이동합니다.

> **참고**: 입주자 계정은 3일 동안 사용 가능합니다. 장기 사용이 필요한 경우 관리자에게 문의하세요.

## 🎯 주요 기능

### 1. 하자 등록

#### 하자 등록 방법
1. 하단 탭바에서 **"➕ 하자 등록"** 탭 선택
2. **하자명** 선택 (드롭다운)
   - 하자명 선택 시 자동으로 설명과 해결방법이 표시됩니다.
   - YouTube 동영상이 자동으로 검색되어 표시됩니다.
3. **위치** 입력 (예: 거실, 주방, 안방)
4. **세부공정** 입력 (예: 바닥재, 타일, 벽지)
5. **하자 내용** 확인 (자동 입력됨, 필요시 수정 가능)
6. **메모** 입력 (선택사항, 현장 특이사항 등)
7. **사진 촬영/업로드**
   - **전체사진**: 하자 위치 전체를 보여주는 사진
   - **근접사진**: 하자 세부를 보여주는 사진
   - 카메라로 촬영하거나 갤러리에서 선택 가능
8. **저장** 버튼 클릭

#### AI 하자 분석 (선택사항)
- 설정에서 **"AI 하자 분석"** 기능을 활성화하면 사진 선택 시 자동으로 하자 유형을 분석합니다.
- 분석 결과는 참고용이며, 최종 하자 유형은 사용자가 선택해야 합니다.

#### YouTube 동영상
- 하자명 선택 시 관련 YouTube 동영상이 자동으로 검색됩니다.
- 동영상을 재생하여 하자 확인 방법을 학습할 수 있습니다.
- 여러 검색 결과 중 원하는 동영상을 선택할 수 있습니다.

---

### 2. 하자 목록 조회

#### 케이스 목록 보기
1. **홈** 탭에서 등록한 케이스 목록을 확인할 수 있습니다.
2. 각 케이스는 다음 정보를 포함합니다:
   - 케이스 ID
   - 등록일시
   - 하자 개수
   - 케이스 유형 (하자접수, 장비점검, 종합점검)

#### 케이스 상세 보기
- 케이스 카드를 클릭하면 상세 정보를 볼 수 있습니다.
- 포함 정보:
  - 하자 목록
  - 사진
  - 등록 일시
  - 메모

---

### 3. 장비 점검 (점검원 전용)

> **참고**: 장비 점검 기능은 **점검원 권한**이 필요합니다. 입주자 계정으로는 접근할 수 없습니다.

#### 점검원 등록
1. 하단 탭바에서 **"점검원등록"** 탭 선택 (입주자만 표시)
2. 다음 정보를 입력:
   - 점검원 이름
   - 전화번호
   - 회사명 (선택사항)
   - 면허번호 (선택사항)
   - 이메일 (선택사항)
   - 등록 사유
3. **등록 신청** 버튼 클릭
4. 관리자 승인 후 점검원 권한이 부여됩니다.

#### 장비 점검 수행
점검원 권한이 승인되면 다음 점검을 수행할 수 있습니다:

##### 열화상 점검
1. **장비 점검** 탭 → **열화상** 탭 선택
2. 위치, 공정, 메모 입력
3. 열화상 이미지 파일 업로드 (jpg, png, 최대 10MB)
4. 결과 선택 (정상/확인요망/해당없음)
5. **저장** 버튼 클릭

##### 공기질 측정
1. **장비 점검** 탭 → **공기질** 탭 선택
2. 위치, 공정, 메모 입력
3. 측정값 입력:
   - **TVOC**: 0-20 mg/m³ (소수점 2자리)
   - **HCHO**: 0-20 mg/m³ (소수점 2자리)
   - **CO₂**: 0-10000 (선택사항)
4. 결과 선택 (정상/확인요망/해당없음)
5. **저장** 버튼 클릭

##### 라돈 측정
1. **장비 점검** 탭 → **라돈** 탭 선택
2. 위치, 공정, 메모 입력
3. 라돈값 입력:
   - **라돈값**: 0-5000
   - **단위**: Bq/m³ 또는 pCi/L 선택
4. 결과 선택 (정상/확인요망/해당없음)
5. **저장** 버튼 클릭

##### 레벨기 측정
1. **장비 점검** 탭 → **레벨기** 탭 선택
2. 위치, 공정, 메모 입력
3. 측정값 입력:
   - **좌측**: -50~+50mm (소수점 1자리)
   - **우측**: -50~+50mm (소수점 1자리)
4. 결과 선택 (이상없음/확인요망)
5. **저장** 버튼 클릭

#### 점검 항목 조회 및 삭제
- 저장된 점검 항목은 케이스 상세 화면에서 확인할 수 있습니다.
- 필요시 점검 항목을 삭제할 수 있습니다.

### 4. 보고서 생성

#### 보고서 미리보기
1. **홈** 탭에서 **"보고서 미리보기"** 버튼 클릭
2. 현재 등록된 모든 하자 및 장비 점검 결과가 포함된 종합 보고서를 확인할 수 있습니다.

#### 보고서 내용
- **하자 정보**: 위치, 공정, 하자 내용, 사진
- **장비 점검 결과**: 
  - 공기질 측정값 (TVOC, HCHO, CO₂)
  - 라돈 측정값
  - 레벨기 측정값 (좌/우)
  - 열화상 사진
- **고객 정보**: 단지명, 동, 호, 성명
- **생성 일시**

### 5. 설정

#### AI 기능 설정
- **AI 하자 분석**: 사진 선택 시 Azure OpenAI로 자동 하자 감지 기능 활성화/비활성화
- 기본값: 비활성화

#### 푸시 알림 설정

##### 푸시 알림 활성화
1. **설정** 화면에서 **"푸시 알림"** 섹션 찾기
2. **푸시 알림** 토글 스위치를 **ON**으로 변경
3. 브라우저에서 알림 권한 요청 팝업이 나타나면 **"허용"** 클릭
4. 알림 상태가 **"활성화됨"**으로 변경되면 설정 완료

##### 알림 유형
푸시 알림을 활성화하면 다음 알림을 받을 수 있습니다:
- **하자 등록 완료 알림**: 하자가 등록되면 관리자에게 알림 발송
- **점검 결과 등록 알림**: 장비 점검이 완료되면 입주자에게 알림 발송
- **점검원 승인/거부 알림**: 점검원 등록 신청 승인/거부 시 알림 발송
- **보고서 생성 완료 알림**: 보고서가 생성되면 해당 사용자에게 알림 발송

##### 테스트 알림
1. **"테스트"** 버튼 클릭
2. 브라우저에서 알림이 수신되는지 확인
3. 알림 클릭 시 앱이 열리는지 확인

##### 푸시 알림 비활성화
- **푸시 알림** 토글 스위치를 **OFF**로 변경
- 더 이상 푸시 알림을 받지 않습니다

> **참고**: 
> - 푸시 알림 사용 시 브라우저 알림 권한이 필요합니다
> - Chrome, Edge, Firefox, Safari (16+) 최신 버전에서 지원됩니다
> - 모바일 브라우저에서도 지원됩니다 (iOS Safari 16+, Chrome Mobile)

#### 앱 정보
- 앱 버전 확인
- 빌드 날짜 확인

## ❓ 문제 해결

### 로그인이 안 될 때
- 단지명, 동, 호가 정확한지 확인하세요.
- 처음 사용하는 경우 관리자에게 계정 등록을 요청하세요.

### 하자 등록이 안 될 때
- 인터넷 연결을 확인하세요.
- 사진 크기가 10MB 이하인지 확인하세요.
- 필수 항목(하자명, 위치, 세부공정)을 모두 입력했는지 확인하세요.

### 장비 점검 탭이 보이지 않을 때
- 점검원 권한이 필요합니다.
- 점검원 등록 신청 후 관리자 승인을 기다려주세요.

### 사진이 업로드되지 않을 때
- 사진 파일 형식이 jpg, png인지 확인하세요.
- 파일 크기가 10MB 이하인지 확인하세요.
- 인터넷 연결을 확인하세요.

### 푸시 알림이 오지 않을 때

#### 1. 브라우저 권한 확인
- **Chrome/Edge**: 
  - 주소창 왼쪽 자물쇠 아이콘 클릭 → 사이트 설정 → 알림 → 허용으로 변경
- **Firefox**: 
  - 주소창 왼쪽 자물쇠 아이콘 클릭 → 권한 → 알림 → 허용
- **Safari**: 
  - Safari → 환경설정 → 웹사이트 → 알림 → 해당 사이트 허용

#### 2. 푸시 알림 설정 확인
- 설정 화면에서 푸시 알림 토글이 **ON**인지 확인
- 알림 상태가 **"활성화됨"**인지 확인
- **"테스트"** 버튼으로 작동 여부 확인

#### 3. 브라우저 호환성 확인
- Chrome, Edge, Firefox 최신 버전 사용
- Safari 16 이상 사용 (iOS Safari 16+)
- 브라우저가 푸시 알림을 지원하는지 확인

#### 4. Service Worker 확인
- 브라우저 개발자 도구 (F12) 열기
- **Application** 탭 → **Service Workers** 선택
- Service Worker가 등록되어 있고 활성화되어 있는지 확인
- 문제가 있으면 **Unregister** 후 페이지 새로고침

#### 5. 인터넷 연결 확인
- 안정적인 인터넷 연결 확인
- 방화벽이나 보안 소프트웨어가 푸시 알림을 차단하지 않는지 확인

#### 6. 계정 재로그인
- 로그아웃 후 다시 로그인
- 푸시 알림 설정 다시 활성화

### YouTube 동영상이 표시되지 않을 때
- 인터넷 연결을 확인하세요.
- 하자명을 다시 선택해보세요.
- 관련 동영상이 없는 경우일 수 있습니다.
