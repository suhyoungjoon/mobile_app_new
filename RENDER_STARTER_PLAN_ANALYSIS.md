# 💰 Render Starter 플랜 분석

## 📊 플랜 비교

| 플랜 | 월 비용 | 빌드 타임아웃 | 메모리 | 디스크 | 빌드 성공 가능성 |
|------|---------|---------------|--------|--------|-----------------|
| **Free** | $0 | 짧음 (~5분?) | 512MB | 1GB | ❌ 실패 (확인됨) |
| **Starter** | $7 | 더 길게 | 512MB | 1GB | ⚠️ **확인 필요** |
| **Standard** | $25 | 충분 | 1GB | 2GB | ✅ 확실 |

---

## 🔍 Starter 플랜 분석

### ✅ 장점
1. **비용 효율**: Standard의 28% 비용 ($7 vs $25)
2. **빌드 타임아웃 증가**: Free보다 더 긴 타임아웃
3. **프로덕션 기능**: 무료 플랜에 없는 기능 제공

### ⚠️ 제한사항
1. **메모리 512MB**: Free와 동일 (Puppeteer 실행에는 충분하지만 제한적)
2. **디스크 1GB**: Free와 동일 (Chromium 300MB + 앱 파일 저장 가능)
3. **빌드 타임아웃**: Standard보다 짧을 수 있음

---

## 🎯 Puppeteer Chromium 다운로드 가능성

### 요구사항
- **Chromium 크기**: 약 300MB
- **다운로드 시간**: 3-5분
- **디스크 공간**: 300MB + 앱 파일 (~100MB) = 약 400MB 필요

### Starter 플랜 여유
- **디스크 1GB**: 충분 (400MB 사용 후 600MB 여유)
- **빌드 타임아웃**: Free보다 길지만 정확한 시간 불명확

### ⚠️ 위험 요소
- **빌드 타임아웃**: Free 플랜에서 실패했으므로, Starter도 실패할 가능성
- **메모리**: 512MB는 충분하지만, 빌드 중 메모리 사용량이 높을 수 있음

---

## 💡 권장사항

### 옵션 1: Starter 플랜 시도 (비용 효율) ⭐

**장점**:
- ✅ 비용 절감 ($7 vs $25)
- ✅ 빌드 타임아웃 증가로 성공 가능성 높음
- ✅ 실패 시 Standard로 쉽게 업그레이드 가능

**단계**:
1. Starter 플랜으로 업그레이드
2. 빌드 테스트
3. 성공 시: Starter 유지
4. 실패 시: Standard로 업그레이드

**예상 결과**:
- 성공 가능성: 60-70% (빌드 타임아웃이 충분히 길다면)
- 실패 시: Standard로 즉시 업그레이드 가능

- ✅ 프로덕션 환경에 최적

**단점**:
- ❌ 비용이 더 높음 ($25 vs $7)

**이유**:

**전략**:
1. Starter 플랜으로 업그레이드
2. 빌드 테스트 (1-2회)
3. 성공 시: Starter 유지
4. 실패 시: Standard로 즉시 업그레이드

## 📋 Starter 플랜 업그레이드 단계

### 1. Render Dashboard 접속
- https://dashboard.render.com
- `insighti-backend-v2` 서비스 선택

### 2. Settings 탭 → Plan 변경
- 현재: Free
- 변경: **Starter** ($7/월) 선택
- Save Changes

### 3. Build Command 확인
- `npm install`로 설정되어 있는지 확인
- (이미 `render.yaml`에 반영됨)

### 4. 환경 변수 확인
- `PUPPETEER_SKIP_CHROMIUM_DOWNLOAD` 제거 (필요 없음)
- 기존 환경 변수 유지

### 5. 재배포
- Manual Deploy 클릭
- 빌드 로그 확인
